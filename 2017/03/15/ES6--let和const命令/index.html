ES6笔记--- const和let

发表于 2017-03-15 | 分类于 ES6学习 |

1.简介

ECMAScript是javascript语言的下一代标准，也称作ES2015

2.let

2.1基本用法

let的用途跟var相似，都是用来声明变量的，但是let声明的变量只在let命令所在的代码块里有效

在for循环中，可以使用let来定义变量，防止变量名冲突

2.2不存在变量提升

上述代码变量foo用var进行声明，存在变量提升，就是预解释，发生在代码执行之前，也就是说代码执行之前，foo已经存在了，而变量bar用let声明，不存在变量提升，在代码从上向下执行的时候bar是不存在的，所以会报错

2.3不允许重复声明

let不允许在相同的作用域内，重复声明同一个变量

同理，不能在函数内部重新声明参数

3.块级作用域

3.1 ES5中的作用域

ES5中只有全局作用域和函数作用域，没有块级作用域，所以会带来很多问题

1)内层变量覆盖外层变量

因为var 有变量提升，函数执行时候，内层web覆盖外层web，所以输出undefined

2)用来计数的循环变量泄漏为全局变量

因为var 有变量提升，函数执行时候，内层web覆盖外层web，所以输出undefined

3.3 块级作用域与函数声明

4.2 const本质

const实际上保证的不是变量的值不可改动，而是变量指向的那个内存地址不得改动，对于简单类型的数据，值就保存在变量指向的那个内存地址，因此等同于常量；
但对于复合类型的数据，变量指向的内存地址，保存的是一个指针，const只能保证这个指针是固定的，至于它指向的数据结构是不是可变的就完全不能控制了。

上面代码中，常量foo存储的是一个地址，这个地址指向一个对象，不可变得只是这个地址，不能把foo指向另一个地址，但是对象本身是可变的
ES6声明变量的方法
ES5 只有两种声明变量的方法：var命令和function命令。ES6除了添加let和const命令，后面章节还会提到，另外两种声明变量的方法：import命令和class命令。所以，ES6 一共有6种声明变量的方法。

5.顶层对象

顶层对象，在浏览器环境指的是window对象，在Node指的是global对象，ES5之中，顶层对象的属性和全局变量是等价的

上面代码中，顶层对象的属性赋值与全局变量的赋值，是同一件事。
ES6中，一方面规定，为了保持兼容性，var命令和function命令声明的全局变量，依旧是顶层对象的属性；另一方面规定，let命令、const命令、class命令声明的全局变量，不属于顶层对象的属性。

6.global对象

ES5的顶层对象，在各种实现里是不统一的

  • 浏览器里面，顶层对象是window，但Node和Web Worker没有window
  • 浏览器和Web Worker里面，self也指向顶层对象，但是Node没有self
  • Node里面，顶层对象是globak，但其他环境都不支持。

同一段代码为了能够在各种环境，都能取到顶层对象，现在一般是使用this变量，但是有局限性

  • 全局环境中，this会返回顶层对象。但是，Node模块和ES6模块中，this返回的是当前模块。
  • 函数里面的this，如果函数不是作为对象的方法运行，而是单纯作为函数运行，this会指向顶层对象。但是，严格模式下，这时this会返回undefined
  • 不管是严格模式，还是普通模式，new Function(‘return this’)()，总是会返回全局对象。
